Problem Set 04¶

In a multi-armed bandit problem, the reward of different arms are assumed to be independent. In our simulated environment, the reward of each arm pull is sampled from a Gaussian distribution with unknown mean and known standard deviation, i.e., $ r_{i,t} \sim \mathcal N (\mu_i, \sigma^2) $, where $ i $ is arm index and $ t $ is the timestamp. The agent can only observe the index of each arm and the sample reward each time; and therefore, the learner should maintain sufficient statistics and construct reward estimator for each arm accordingly.

Implement the explore-and-commit algorithm: The detailed procedure of this algorithm can be found below. The only hyper-paramater we need to choose is the length of the exploration phase (denoted as $ m $), whose theoretical value has also been discussed in the slides. Please implement this algorithm following steps discussed in class (Part A, 50 pts).¶

Explore-Then-Commit Algorithm¶

This implementation follows the Explore-Then-Commit strategy for multi-armed bandits that we discussed in class. The objective is to minimize regret $R_T = T\mu^* - \mathbb{E}\left[\sum_{t=1}^T X_t\right]$, where $\mu^* = \max_i \mu_i$ is the highest true mean among all arms.

Concepts from Slides & Framework¶

The ETC algorithm operates in two phases:

  1. Exploration Phase: Each arm is pulled $m$ times, and empirical means $\hat\mu_i$ are computed.
  2. Commit Phase: The arm with the highest empirical mean is chosen for all remaining rounds.

The regret decomposes into: $$ R_T \leq m \Delta + T \Delta \exp\left(-\frac{m\Delta^2}{4}\right) $$ where $\Delta = \mu^* - \mu_{(2)}$ is the gap between the best and second-best arm. This bound captures the trade-off between:

  • Exploration Regret ($m \Delta$): Increases with $m$.
  • Exploitation Regret ($T\Delta \exp(-m\Delta^2/4)$): Decreases with $m$.

To balance this trade-off, we use the theoretically motivated: $$ m = \left\lceil \frac{4}{\Delta^2} \log\left(\frac{T \Delta^2}{4}\right) \right\rceil $$ If $T\Delta^2 / 4 < 1$, we fall back to $m = \lfloor T / (2K) \rfloor$ to avoid taking the log of a value less than 1.

Implementation Details¶

  • For each $T$ and $K$, we generate a set of true means from $\text{Uniform}(0, 1)$.
  • We compute $\Delta$ from the sorted true means (only well-defined for $K = 2$).
  • During exploration, each arm is pulled $m$ times and empirical means $\hat\mu_i$ are computed.
  • In the commit phase, the arm with the highest $\hat\mu_i$ is pulled for the remaining $T - mK$ rounds.
  • We calculate:
    • Total reward accumulated.
    • Regret compared to the optimal arm.
    • The theoretical regret bound for comparison.

This approach is grounded in the assumption that the reward distributions are $\sigma$-subgaussian, allowing us to apply the exponential concentration bound that controls the probability of choosing a suboptimal arm.

Limitations Acknowledged¶

  • The value of $\Delta$ is unknown in practice and estimated using observed $\mu_i$.
  • For $K > 2$, defining $\Delta$ becomes more complex and generalizing the regret bound requires pairwise gap considerations.
  • The theoretical $m$ is optimal only under the idealized assumptions of the regret bound.

def explore_then_commit(T, K, true_means, sigma):
    
    # Compute the gap Delta (suboptimality gap) between the best and second-best arm. 
    # We do the second best to fulfill the \mu^* - \mu_i where the only i in the K = 2 case is \mu_2.
    sorted_means = np.sort(true_means)[::-1]
    Delta = sorted_means[0] - sorted_means[1]
    
    # Compute m using the formula from the slides.
    # But if T * Delta^2 / 4 < 1, the log becomes negative.
    # In that case, we set m to a fallback value.
    if (T * (Delta**2)) / 4 < 1:
        m = T // (2 * K)
    else:
        m = int(np.ceil((4 / (Delta**2)) * np.log((T * (Delta**2)) / 4)))
    
    # Ensure m is at least 1 and at most T/K.
    m = max(1, m)
    m = min(m, T // K)
    
    # Initialize lists to store rewards and chosen arms.
    rewards = []
    chosen_arms = []
    
    # Dict to store rewards for each arm during exploration.
    arm_rewards = {i: [] for i in range(K)}
    
    # ========== EXPLORATION PHASE ==========
    # Pull each arm m times.
    for i in range(K):
        for _ in range(m):
            reward = np.random.normal(true_means[i], sigma)
            arm_rewards[i].append(reward)
            rewards.append(reward)
            chosen_arms.append(i)
    
    # Compute the empirical means for each arm.
    empirical_means = np.array([np.mean(arm_rewards[i]) for i in range(K)])
    
    # Identify the best arm based on the empirical means.
    best_arm = np.argmax(empirical_means)
    
    # ========== COMMIT PHASE ==========
    remaining_rounds = T - (m * K)
    for _ in range(remaining_rounds):
        reward = np.random.normal(true_means[best_arm], sigma)
        rewards.append(reward)
        chosen_arms.append(best_arm)
    
    # ========== REGRET CALCULATION ==========
    best_true_mean = np.max(true_means)
    total_optimal_reward = best_true_mean * T
    total_reward = np.sum(rewards)
    regret = total_optimal_reward - total_reward

    # ========== THEORETICAL REGRET BOUND ==========
    theory_regret = m * Delta + T * Delta * np.exp(-m * (Delta**2) / 4)
    
    return {
        "total_reward": total_reward,
        "regret": regret,
        "theory_regret": theory_regret,
        "exploration_empirical_means": empirical_means,
        "chosen_best_arm": best_arm,
        "chosen_arms_history": chosen_arms,
        "rewards_history": rewards,
        "exploration_rounds_per_arm": m
    }

def run_multiple_simulations(T, K, sigma, num_runs=1000):
    """
    Runs the explore-then-commit simulation multiple times and plots
    the running average of empirical vs. theoretical regret.
    """
    empirical_regrets = []
    theoretical_regrets = []
    
    for _ in range(num_runs):
        
        # Generate new random means for each run
        true_means = np.random.uniform(0, 1, size=K)
        
        result = explore_then_commit(T, K, true_means, sigma)
        empirical_regrets.append(result["regret"])
        theoretical_regrets.append(result["theory_regret"])
    
    # Compute running averages
    cumulative_avg_empirical = np.cumsum(empirical_regrets) / np.arange(1, num_runs+1)
    cumulative_avg_theoretical = np.cumsum(theoretical_regrets) / np.arange(1, num_runs+1)
    
    # Plot
    plt.figure(figsize=(10, 6))
    plt.plot(cumulative_avg_empirical, label='Empirical Regret (Running Avg.)')
    plt.plot(cumulative_avg_theoretical, label='Theoretical Bound (Running Avg.)')
    plt.xlabel('Number of Simulation Runs')
    plt.ylabel('Regret')
    plt.title('Explore-Then-Commit: Empirical vs. Theoretical Regret')
    plt.legend()
    plt.show()

    # Return the final averages if you want them
    return cumulative_avg_empirical[-1], cumulative_avg_theoretical[-1]

        m_candidates = range(1, T // K + 1)
        
    results = {}

        results[m] = avg_regret
